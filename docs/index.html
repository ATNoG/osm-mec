Introduction

Welcome to OSM-MEC. In a world dominated by data consumption, efficiency and low-latency computing are non-negotiable. That's where Multi-access Edge Computing (MEC) steps in.

Our focus is clear - to bridge the gap between MEC and Network Functions Virtualization (NFV) We will develop a MEC Application Orchestrator (MEAO), along with a MEC-platform for showcasing purposes and a monitoring platform for bothe MEC and MEC applications.

Our Work

Our Work will consist on the developement of the System Level of Multi-access Edge Computing. This system is composed by three main components:

  • OSS (Operations Support System)
  • MEAO (MEC Application Orchestrator)
  • User application lifecycle management proxy (not MVP)

The MEAO is the core of MEC's System level Management. It's responsible for tasks such as processing information about the MEC apps (such as application packages and their integratity and authenticity, application rules, requirements, etc...), translating MEC functionalities onto the corresponding VNF's, selecting appropriate MEC hosts according to constraints and trigger instantiation of the apps.

The OSS is the system that receives requests from device applications or the portal for instantiation and termination of MEC apps and decides wether these requests are accepted, forwarding them if so to the MEAO. It can also receive requests for relocating applications between external clouds and the MEC system.

OSM stands for Open Source MANO (Management and Orchestration). This is a open-source ETSI-hosted project dedicated to providing a robust Management and Orchestration stack for NFV, capable of consuming information models accessible to everyone and suitable for all Virtual Infrastructure Managers (VIM).

About Us

Our work was born in a proposal for Projeto em Engenharia de Computadores e Informática (PECI) course project made by Professor Diogo Gomes. It's noticeable that we only developped a MEAO, which is a part of the MEC architecture. The rest of the MEC platform was done by students from the Masters Degree in Computers and Telematics Engineering, as well as one Phd student, all guided by Prof. Diogo.

What made us choose this proposal was the fact that this project was (in our opinion) more relevant than the others in terms of value to society. Not only that, we actually like the area of Networking.
